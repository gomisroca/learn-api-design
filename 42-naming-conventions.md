# Naming Conventions

- All names used in the API **should** be straightforward, intuitive, and consistent.
- All names **should** be in English.
- Short forms **may** be used for commonly-used names.
- Definitions **must** use UpperCamelCase.
- Intuitive, familiar terminology where possible.
- Same name for the same concept, shared across the API.
- Avoid name overloading.
- Avoid overly general names.

### Interface Names

- An interface name **should** use an intuitive noun.
- The name **should not** conflict with well-known concepts.

### Method Names

- A service **may** define one or more API methods.
- Names **should** follow the VerbNoun pattern in UpperCamelCase.

### Message Names

- Message names **should** be short and concise.
- Message names **should not** include prepositions.
