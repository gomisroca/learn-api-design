# Common Components

- CC packages **must** end with .type
- **Must** consult the API design team before adding a new CC package.
- **Must not** define generic components in org-specific CC packages.
- CCs **must not** be moved.
- Fields **should not** be added to existing messages.
- Values **should not** be added to existing enums.
- Fileds **must not** be removed from existing messages.
- Values **must not** be removed from existing enums.
- Documentation **may** be clarified, but it **should not** change the meaning of existing values.
